üå± Greenfield Development Process

Iterative story implementation with clean context management between agents.

‚ö†Ô∏è Critical Context Management:

ALWAYS start a new chat between SM, Dev, and QA work to maintain clean context windows!

1
Story Creation

Start NEW CLEAN CHAT - Create user stories from sharded PRD documents.

Commands:

# For sharded PRD (most common) @sm *create # For brownfield without sharding @sm # Then use brownfield-create-story task

üìÑ Document Output: story-1.1.md in docs/stories/

Status: Draft ‚Üí Review & approve ‚Üí Approved

Story Structure:

  • User story (As a... I want... So that...)
  • Acceptance criteria
  • Tasks/Subtasks checklist
  • Dev notes with technical context

2
Story Implementation

Start NEW CLEAN CHAT - Implement the approved story with file operations and testing.

Commands:

@dev # Agent asks which story to implement # Provide story file content or path

‚úÖ Dev Actions:

  • Implement all tasks/subtasks
  • Create/modify files
  • Write tests
  • Update File List in story
  • Mark story as "Review" when complete

3
QA Review (Recommended)

Start NEW CLEAN CHAT - Quality assurance review and testing.

Commands:

@qa review-story docs/stories/story-1.1.md

‚úÖ QA Actions:

  • Senior dev code review
  • Fix small issues directly
  • Leave checklist for remaining items
  • Update story status

If QA leaves unchecked items, return to Dev (new chat) to address them.

4
Deployment & Release

Deploy completed features using various deployment strategies.

Build Commands:

@devops build --environment staging @devops build --environment production

Deployment Strategies:

# Blue-Green Deployment @devops deploy --strategy blue-green --environment production --version 1.0.0 # Canary Deployment (10% traffic) @devops deploy --strategy canary --feature USER-001 --percentage 10 # Rolling Deployment @devops deploy --strategy rolling --environment staging --version 1.0.0

Monitoring & Release:

@devops release --environment production --version 1.0.0 --strategy blue-green @devops monitor --environment production --alerts-enabled

5
Repeat Cycle

Continue the SM ‚Üí Dev ‚Üí QA ‚Üí DevOps cycle for each story until all epic stories are complete.

‚ö†Ô∏è Remember:

Only 1 story in progress at a time. Complete each story before starting the next!

üí° Pro Tips & Best Practices

‚úÖ Best Practices

  • Keep PRD and Architecture docs as single source of truth
  • Let agents handle their specialized roles
  • Story files are the communication medium between agents
  • Commit frequently to track AI-generated changes
  • Use clean context management between agent phases
  • Test deployments with canary releases before full rollout

‚ö†Ô∏è Common Pitfalls to Avoid

  • ‚Ä¢ Don't skip the planning phase
  • ‚Ä¢ Don't mix agent roles
  • ‚Ä¢ Don't forget to shard documents before development
  • ‚Ä¢ Don't modify story files manually
  • ‚Ä¢ Don't deploy to production without proper testing
  • ‚Ä¢ Don't ignore rollback procedures
